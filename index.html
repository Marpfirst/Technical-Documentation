Python and Ruby

Python and Ruby share some similarities but also have fundamental differences. Python emphasizes code readability and simplicity, while Ruby focuses on developer happiness and expressiveness.

Unlike Python, which enforces indentation for code blocks and utilizes significant whitespace, Ruby uses the "end" keyword to denote the end of blocks and follows a more flexible indentation style.

In contrast to Python's usage of whitespace and indentation to indicate code structure, Ruby uses "do" and "end" to encapsulate blocks of code, enhancing its readability in certain contexts.

While Python uses a global interpreter lock (GIL), limiting its concurrency, Ruby offers better concurrency support through various concurrency models and allows developers to choose among them based on their needs.

Variable scope

In both Python and Ruby, variable scope is determined by where the variable is defined. Variables declared within a block are accessible within that block and can have different scopes based on context. 

# Python example
def example_function():
    if True:
        x = 5
    print(x)  # Outputs 5

example_function()

# Ruby example
if true
  x = 5
end
puts x  # Outputs 5
